Verification Case

嵌套命名空间是一个非常小的写法改进，如：

1

4

// Style1
namespace A { namespace B { namespace C {

}}}

或：

// Style2
namespace A {
namespace B {
namespace C {

}
}
}

或：

1

3
4
5

7
8

// Style3
namespace A {
  namespace B {
    namespace C {

    }
  }
}

可以写成：

1
2
3

namespace A::B::C {

}

较少{}的嵌套着实提升了代码的简洁性。但从个人的角度，依然不会选择这种写法：

  1. 多数代码不会采用Style3的命名空间风格，Style1/Style2并不存在缩进的问题
  2. 命名空间嵌套数量通常在1-3之间，少数会有4层嵌套，所以命名空间嵌套大多不复杂
  3. 显示使用namespace更为醒目，以及方便关键字查找，且::通常不用于定义，而用于使用
  4. 没有可观的收益，且增加额外的概念，不如丢弃
